Stage 1 Notes

Work Session 1

HTML is a web development language that creates structure for web pages. HTML creates structure in such a way that a human reading HTML code can visually see the structure or a browser interacting with the code can easily interpret it. HTML accomplishes this by breaking content down into components, and then those components are further broken down into their sub-components. This model of division and classification is visually represented for humans in the HTML code's tree-like structure, and it is also represented visually by boxes around content dispalyed by the browser. This model of classification is has technologial empathy becuase its systematic organization helps browsers to interact with it. How all of that works, I'm not sure at all!

Work Session 2

CSS is a web development language that applies style to web pages. CSS, cascading style sheets, applies style rules to HTML elements. Often a single element will have many style rules apply to it. The style rule that is the most specitic to the element "wins" and that is the style that the browswer ultimately applies.

Like HTML, CSS relies on "boxifying" elements on a page. With CSS you can manipulate the size and position of these boxes to suite your design.
